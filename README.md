# Road Sign Detector (YOLOv11)

Програмний додаток для розпізнавання дорожніх знаків на фото та відео.
Розроблено в рамках дипломної роботи.

## 🧠 Технічні деталі (Модель та Дані)
- **Модель:** Проєкт базується на сучасній архітектурі **[YOLOv11](https://github.com/ultralytics/ultralytics)** від компанії **Ultralytics**. Файл `best.pt` містить ваги моделі, отримані шляхом донавчання під задачу детекції знаків.
- **Датасет:** Для навчання використано набір даних із платформи комп'ютерного зору **Roboflow** (ознайомитися з даними можна за посиланням: [Traffic Signs Detection](https://universe.roboflow.com/radu-oprea-r4xnm/traffic-signs-detection-europe/dataset/14)). Зображення розмічені у форматі YOLOv11.
- **Обмеження:** Стандартна роздільна здатність для інференсу становить **640x640 пікселів**. Усі вхідні файли та потоки з веб-камери автоматично масштабуються до цього розміру із збереженням пропорцій, щоб уникнути спотворень і втрати точності.

## ⚙️ Функціонал
- 📸 **Фото:** Детекція знаків на статичних зображеннях.
- 🎬 **Відео:** Обробка відеофайлів (.mp4, .avi).
- 🎥 **Live:** Робота з веб-камерою в реальному часі.
- 💾 **Збереження:** Автоматичне збереження результатів (фото + текстовий звіт).

## Стандарти документування коду

У цьому проєкті використовується стандарт документування **Google Style Docstrings** для Python. 
Він був обраний через свою читабельність та ідеальну сумісність з інструментами автогенерації.

**Правила для розробників:**
1. Кожен новий клас або метод повинен містити `docstring` у потрійних лапках `"""`.
2. Обов'язково описувати призначення функції.
3. Використовувати секції `Параметри:` (Args) та `Повертає:` (Returns) для методів, що приймають або повертають значення.
4. Документація генерується автоматично за допомогою бібліотеки `pdoc`.
